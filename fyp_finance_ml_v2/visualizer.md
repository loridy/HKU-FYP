# visualizer.py - Figure and Summary Generation

## Overview

`visualizer.py` turns numeric outputs from the pipeline into plots and summary figures.

It is the presentation layer of the project: the part that creates the images mentioned in the README, including the final four-panel summary figure.

This module does not train models or compute metrics. It simply visualizes what the rest of the pipeline has already produced.

---

## Plotting Setup

The file sets Matplotlib to use the non-interactive `Agg` backend:

```python
matplotlib.use("Agg")
```

### Why this matters

This makes the module safe to use in headless CLI environments where no GUI display is available.

That matches the project’s batch-oriented execution style.

---

## Main Functions

### 1. `save_equity_curve(df, out_path, title)`

This saves a line plot of cumulative strategy equity.

### Inputs

- `df`: DataFrame with at least `date` and `net_ret`
- `out_path`: file path for the PNG output
- `title`: plot title

### How it works

It computes equity as:

```python
equity = (1 + df["net_ret"]).cumprod()
```

Then it plots equity over time.

### What it is used for

This is how the pipeline writes the per-strategy equity curve plots under `outputs/figures/`.

### Behavior on empty data

If the DataFrame is empty, the function returns immediately and does not write a file.

---

### 2. `save_heatmap_table(pivot_df, out_path, title)`

This writes a matrix-style heatmap with cell labels.

### Typical input

A pivot table such as Rank IC by:

- feature set on the rows
- horizon on the columns

### How it works

- uses `plt.imshow()` for the heatmap
- writes the column and row labels
- writes formatted numeric values directly onto the cells
- adds a colorbar

### Why it matters

This supports the heatmap output described in the README and generated by `pipeline.py`.

---

### 3. `save_four_panel_summary(metrics_df, backtest_df, decile_df, strategy_curves, out_path, main_model="logistic_regression")`

This is the main summary-figure function.

It creates a 2x2 figure with the following panels:

1. Rank IC heatmap
2. Cost-adjusted Sharpe bar chart
3. Equity curve comparison
4. Bucket forward return profile

### Panel A: Rank IC heatmap

This panel uses `metrics_df` filtered to the selected main model and pivots by feature set and horizon.

### Panel B: Cost-adjusted Sharpe

This panel uses `backtest_df` and displays Sharpe across feature sets and horizons.

### Panel C: Equity curves

This panel plots the cumulative equity curves for the strategy curves passed in via `strategy_curves`.

The pipeline typically includes:

- best ML strategy
- SPY benchmark
- momentum baseline

### Panel D: Bucket return profile

This panel uses `decile_df` and plots average forward return by bucket.

### Why this figure matters

This summary is the compact, report-ready view of the experiment. It aligns exactly with the README’s statement that the pipeline writes a four-panel one-page summary figure.

---

## File Relationships

### Inputs from other modules

- `pipeline.py` provides `metrics_df`, `backtest_df`, `decile_df`, and selected strategy curves
- `backtest.py` provides daily net return series that become equity curves
- `evaluation.py` provides the metrics used in the heatmap and bucket profile

### Outputs

The module writes PNG files to disk under the configured output directory.

---

## Design Choices

### 1. Headless-safe

Using Agg makes the module portable across local machines, notebooks, and CI-like environments.

### 2. Fail-soft behavior

If a DataFrame is empty, the function returns without raising.

### 3. Compact summary format

The four-panel layout is intended to give a quick but complete view of the experiment.

### 4. Project-consistent labels

The figure titles and panel labels reflect the project’s metric vocabulary: Rank IC, Sharpe, equity curve, and bucket profile.

---

## How It Is Used in the Pipeline

`pipeline.py` calls:

- `save_equity_curve()` for each strategy run
- `save_heatmap_table()` for the Rank IC summary
- `save_four_panel_summary()` for the final report figure

So this file is the final presentation step after all modeling and evaluation work is done.

---

## Summary

`visualizer.py` is the project’s **reporting layer**.

It converts the results of the pipeline into saved figures that are suitable for analysis, presentation, or inclusion in a report.
